# Prompt for outside chatbots

Paste this verbatim into ChatGPT, Gemini, Grok, or any other model.
Attach `REVIEW_PACKAGE.md` (one file) when uploading.

---

I'm reviewing a new Claude Code skill I built called `mad-research`.
The attached file is a complete snapshot. New public GitHub:
<https://github.com/tjhavranek/mad-research> (you may not be able to
find it; everything you need is in the attachment).

**I wanted two things in one package:**

1. Make it easy for any Claude Code user (who also has a basic
   ChatGPT subscription, i.e. Codex CLI) to invoke Codex from Claude
   for any project — independent drafts, cross-feedback, synthesis.
2. Use the same machinery to automate my "MAD-research" idea: a
   structured adversarial audit of a paper, grant, or referee
   report, with quote-grounded criticism, minority report
   preservation, locked rubric, and an audit trail.

The skill should be **easy and robust for non-programmers** — anyone
who knows how to use Claude Code should be able to install and run
it without writing code.

**Please give me only feedback you are confident about and that I
might genuinely act on.** Two or three points maximum. Don't pad.
If you have nothing important to say, say so.

Specifically:

1. Does this skill achieve what I want, both modes, in a usable way?
2. Is this the most parsimonious solution, or am I missing an
   obvious simpler design?
3. Are there cutting-edge MAD / multi-agent-debate research results
   from the last 12 months (consensus dynamics, role design,
   blinding, calibration, judge selection) that this design
   noticeably ignores? If so, name them.
4. Anything else that is important and you are sure about.

Brevity is the test. Three short paragraphs is plenty.
